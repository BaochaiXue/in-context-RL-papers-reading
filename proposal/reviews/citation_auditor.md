# Citation Auditor Review

## Findings

1. **[High] Section 2 turns method-specific limitations into a universal claim that the field lacks a "context compiler."** In [main.tex](/Users/xinjiezhang/in context RL/proposal/main.tex#L51) the text says long context is not enough and that "the missing object is not simply 'more memory,' but a context compiler." The local notes support only narrower statements: [Retrieval-Augmented Decision Transformer](/Users/xinjiezhang/in context RL/notes/papers/retrieval-augmented-decision-transformer-external-memory-for-in-context-rl.md#L15) says retrieval helps but memory and learning remain confounded, [Filtering Learning Histories Enhances In-Context Reinforcement Learning](/Users/xinjiezhang/in context RL/notes/papers/filtering-learning-histories-enhances-in-context-reinforcement-learning.md#L15) says filtering improves several backbones but is still heuristic, and [LMAct](/Users/xinjiezhang/in context RL/notes/papers/lmact-a-benchmark-for-in-context-imitation-learning-with-long-multimodal-demonstrations.md#L15) shows long multimodal contexts still do not reliably produce expert-level behavior. None of those notes establish that raw trajectories are the wrong general interface or that a principled compiler is absent from the literature.

2. **[High] Pillar I overstates what the mechanistic papers justify about internal state.** In [main.tex](/Users/xinjiezhang/in context RL/proposal/main.tex#L71) the proposal jumps from "local notes" to a four-part typed state `z_t=(v_t,u_t,b_t,c_t)` and a claim about the "minimal typed state" needed for near-optimal adaptation. The local evidence is much weaker: [Transformers Learn Temporal Difference Methods for In-Context Reinforcement Learning](/Users/xinjiezhang/in context RL/notes/papers/transformers-learn-temporal-difference-methods-for-in-context-reinforcement-learning.md#L14) is explicitly limited to policy evaluation and synthetic tasks, [Sparse Autoencoders Reveal Temporal Difference Learning in Large Language Models](/Users/xinjiezhang/in context RL/notes/papers/sparse-autoencoders-reveal-temporal-difference-learning-in-large-language-models.md#L14) says the detected features may be proxy mechanisms rather than a full explanation, and [Learning How to Infer Partial MDPs for In-Context Adaptation and Exploration](/Users/xinjiezhang/in context RL/notes/papers/learning-how-to-infer-partial-mdps-for-in-context-adaptation-and-exploration.md#L14) depends on a small hypothesis space plus representation supervision. Together these support "there are RL-like latent quantities," not the specific four-way decomposition or the claim that it is minimal.

3. **[Low] Bibliography hygiene is incomplete: three entries are unused.** `references.bib` includes `nikulin2024xland100b`, `song2025reward`, and `wang2025omnirl`, but none of those keys appear in `main.tex`. I did not find any missing bibliography keys or undefined citations, so this is not a build failure; it is a consistency issue that suggests either stale bib entries or text claims that never received citations.

## What to fix

- Rephrase the literature review as a set of observed limitations of specific methods, not as proof that the field lacks a principled compiler.
- Recast Pillar I as a testable hypothesis about candidate latent variables, then tie it to explicit intervention metrics instead of a predeclared four-slot state.
- Either remove the three unused bibliography entries or add the text that cites them.
