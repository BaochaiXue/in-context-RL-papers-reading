- **High:** The evaluation logic is not yet concrete enough to de-risk the core claims. Pillar I says it will compare typed-state models under counterfactual edits, but it does not define the intervention metric, the success threshold, or how to tell an explicit state from a useful latent proxy ([`main.tex`](/Users/xinjiezhang/in context RL/proposal/main.tex#L75)). Pillar II says the compiler will be judged by whether memory is "task-sufficient," but there is no retention test, invalidation test, or ablation protocol beyond downstream performance ([`main.tex`](/Users/xinjiezhang/in context RL/proposal/main.tex#L95)). Pillar III lists calibration, regret after shifts, intervention frequency, and catastrophic failure rate, but the proposal does not specify how these are traded off or when abstention is allowed ([`main.tex`](/Users/xinjiezhang/in context RL/proposal/main.tex#L111)). The notes show why this matters: performance depends heavily on history availability, coverage, and prompt scaffolding, so without fixed evaluation rules the results will be hard to interpret ([`yes-q-learning-helps-offline-in-context-rl.md`](/Users/xinjiezhang/in context RL/notes/papers/yes-q-learning-helps-offline-in-context-rl.md#L14), [`random-policy-enables-in-context-reinforcement-learning-within-trust-horizons.md`](/Users/xinjiezhang/in context RL/notes/papers/random-policy-enables-in-context-reinforcement-learning-within-trust-horizons.md#L14), [`can-large-language-models-explore-in-context.md`](/Users/xinjiezhang/in context RL/notes/papers/can-large-language-models-explore-in-context.md#L14), [`evolve-evaluating-and-optimizing-llms-for-exploration.md`](/Users/xinjiezhang/in context RL/notes/papers/evolve-evaluating-and-optimizing-llms-for-exploration.md#L14)).

## What to fix

Narrow the proposal to one primary pillar and one secondary validation thread, then rewrite the timeline around the actual data and compute you already have. For each remaining experiment family, add a concrete success metric, a failure criterion, and a fallback dataset or environment so the plan can be executed without assuming AdA/LMAct-level infrastructure from the start.
